## 事故概述

!!! warning "页面状态：占位稿（待完善）"
    本页为结构化占位稿，内容用于版式示例

2012年7月30日至31日，印度连续两天发生大规模停电，影响约6.2亿人，占当时世界人口约9%，是人类历史上影响人数最多的停电事故。

与多数由单点设备故障触发的大停电不同，印度此次事故的核心是长期结构性矛盾集中爆发，即供需失衡、电网基础设施薄弱、部分邦长期超额取电，再叠加季风期间农业灌溉负荷上升，最终触发跨区域连锁崩溃。

---

## 事故根源分析

### 直接原因

- 北方电网（Northern Grid）多条500kV输电线路相继过载跳闸
- 部分邦（尤其北方邦、哈里亚纳邦）显著超出分配用电配额
- 电网频率持续偏低，系统长期处于脆弱运行边缘

### 深层原因

=== "供需层面"

    - **供需长期失衡**：峰值供电缺口约 9%，电力短缺长期化。
    - **季风影响叠加**：2012 年季风偏弱，农业灌溉负荷显著上升。
    - **旋转备用不足**：系统可调用备用能力不足，扰动后难以及时托底。

=== "技术层面"

    - **电网基础设施薄弱**：部分输电线路与变电设备老化，运行裕度偏低。
    - **输配电损耗偏高**：技术损耗与非技术损耗并存，削弱有效供给。
    - **区域互联支撑有限**：区域间紧急功率支援能力不足。

=== "管理层面"

    - **配额约束执行不足**：超额取电长期存在，事前纠偏机制偏弱。
    - **跨区协调调度不足**：多区域协同控制能力与预案执行存在短板。
    - **行政与调度目标冲突**：非技术因素影响实时调度决策。

## 事故时序

### 7月30日（第一次）

```text
02:33  北方电网频率开始异常下降
       多个邦持续超额取用电力
       系统频率跌破安全下限
       北方电网与东部、东北部电网解列
       约4亿人断电，持续约13.5小时
```

### 7月31日（第二次）

```text
13:00  北方、东部、东北部电网再次崩溃
       触发机理与前一日高度相似
       约6.2亿人断电（史上最大规模）
       印度政府启动紧急应对
```

## 影响与后果

### 受影响规模

| 指标 | 数值 |
| ---- | ---- |
| 受影响人口 | 6.2亿（约占当时世界人口9%） |
| 受影响邦数 | 22个邦 |
| 损失发电容量 | 约32 GW |
| 停运列车 | 约300列 |
| 被困矿工 | 约200人（煤矿提升机停运） |

### 次生影响

- 德里地铁大范围停运，乘客滞留
- 医院、铁路、通信系统依赖备用电源维持关键功能
- 多地交通信号失效，城市交通严重拥堵
- 工商业生产中断，社会运行效率显著下降

---

## 印度电力系统结构性问题

!!! warning "这不是一次意外，而是积累已久的系统性危机"

此次事故揭示了印度电力系统在供给、网络、治理三方面的深层矛盾：

1. **装机与需求错配**：需求增长快于有效供给能力提升
2. **高损耗运行常态化**：输配电损耗和非技术损耗并存
3. **区域互济能力不足**：跨区紧急支援能力有限
4. **农业电价扭曲**：补贴政策导致局部负荷刚性上升
5. **调度约束执行不强**：配额制度与惩戒机制存在短板

---

## 事后改革

!!! success "印度电力系统改革提速"

- **强化配额执行**：对超额取电行为实施更严格约束
- **推进智能化改造**：加大智能计量和配网自动化投资
- **扩大可再生能源规模**：以新能源补充供给缺口
- **加强跨区互联**：提升区域间联络和支援能力
- **治理高损耗问题**：持续推进反窃电与损耗管理

---

## 电力系统安全分析视角

| 知识点 | 在本事故中的体现 |
| ------ | ---------------- |
| 频率稳定性 | 长期低频运行使系统稳定裕度持续收窄 |
| 旋转备用 | 扰动后缺乏足够托底能力，恢复困难 |
| 负荷管理 | 配额失效导致高峰期过载风险放大 |
| 区域电网互联 | 跨区支援能力不足限制故障隔离与救援 |
| 供需平衡 | 结构性供需矛盾是事故根本驱动因素 |
