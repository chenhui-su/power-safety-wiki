## 事故概述

!!! warning "页面状态：占位稿（待完善）"
    本页内容用于后续补充，暂不视为最终定稿。

2003年8月14日下午，美加互联电网在高温高负荷条件下发生连锁故障。事故在数小时内由局部线路与机组异常，迅速演化为跨区域大停电，3分钟内约61,800MW负荷丢失。

事故暴露出“设备运行边界逼近 + 调度态势感知失效 + 跨区协调不足”的复合风险。其后北美电力行业对可靠性标准、树障管理和调度技术体系进行了系统性整改。

---

## 事故根源分析

### 直接原因

- Eastlake 5号机组跳闸后区域无功支撑减弱，电压稳定裕度下降
- 多条345kV线路受树障或过载影响相继跳闸，潮流快速重分布
- 控制中心 EMS 报警功能失效，调度员未能及时识别风险升级

### 深层原因

=== "供需层面"

    - **夏季高负荷运行**：高温下空调负荷显著增加，系统运行裕度缩小。
    - **无功储备偏紧**：关键区域动态无功支撑不足，电压风险累积。
    - **实时风险滚动评估不足**：负荷变化与线路风险联动研判不充分。

=== "技术层面"

    - **EMS 关键功能异常**：报警/状态估计失效削弱态势感知。
    - **输电通道树障治理不到位**：线路对地净空管理执行不足。
    - **保护与安全控制协同不足**：连锁故障快速传播，隔离和切负荷不及时。

=== "管理层面"

    - **跨区沟通机制不足**：控制区间告警与协同处置效率不高。
    - **应急训练不足**：对复杂级联场景的组织应对能力有限。
    - **可靠性规则刚性不足（事故前）**：部分标准缺乏强制执行力度。

## 事故时序

### 8月14日（主故障时段）

```text
13:31  FirstEnergy 的 Eastlake 5号机组跳闸
14:02  Chamberlin-Harding 345kV 线路因树障故障跳闸
15:05  Hanna-Juniper 345kV 线路过载跳闸
15:32  Star-South Canton 345kV 线路过载跳闸
15:41  连锁故障开始，多线路相继退出
16:10  大规模停电形成，3分钟内丢失约61,800MW负荷
```

### 8月14-18日（恢复阶段）

```text
8月14日晚  多区域启动黑启动与分区恢复
8月15日    大城市核心负荷逐步恢复
8月16-18日 部分地区持续检修与分段送电，系统恢复常态
```

## 影响与后果

### 受影响规模

| 指标 | 数值 |
| ---- | ---- |
| 受影响人口 | 约5500万人 |
| 受影响区域 | 美国8州 + 加拿大安大略省 |
| 损失发电容量 | 约61.8 GW |
| 关键系统影响 | 地铁瘫痪、机场停摆、通信受扰 |
| 经济影响 | 约60亿美元（公开估算） |

### 次生影响

- 纽约地铁大范围停运，大量乘客滞留
- 航空运输受阻，航班取消与延误增加
- 通信、供水等城市基础服务短时受冲击
- 事故后北美可靠性治理框架被重塑

---

## 北美电力系统结构性问题

!!! warning "这不是一次意外，而是积累已久的系统性危机"

事故揭示了互联大电网在高耦合运行中的系统短板：

1. **态势感知依赖单点系统**：调度核心软件失效会显著放大风险
2. **线路通道治理执行不均衡**：树障管理不到位会触发大风险
3. **跨区协同响应不足**：区域接口处置效率影响连锁传播范围
4. **无功与电压稳定治理不足**：高负荷下电压脆弱性更突出
5. **可靠性标准刚性不足（事故前）**：标准执行缺乏统一强制性

---

## 事后改革

!!! success "北美电力系统可靠性治理提速"

- **强制化可靠性标准**：NERC 可靠性规则由“建议”转向“强制执行”
- **强化树障管理**：提升输电走廊巡检与通道净空治理力度
- **调度系统升级**：加强 EMS/SCADA 冗余与告警可靠性设计
- **跨区协调机制完善**：优化控制区间信息共享与应急联动
- **常态化演练评估**：提升高影响低概率故障场景处置能力

---

## 电力系统安全分析视角

| 知识点 | 在本事故中的体现 |
| ------ | ---------------- |
| 电压稳定 | 无功不足叠加高负荷导致电压脆弱性上升 |
| 态势感知 | EMS 报警失效削弱实时风险识别能力 |
| 连锁故障机理 | 线路跳闸后潮流重分布引发级联扩散 |
| 负荷恢复与黑启动 | 分区恢复能力决定停电持续时间 |
| 可靠性治理 | 事故推动标准体系从建议型转向强制型 |
