## 👥 角色与权限

### 权限边界（当前版本）

- 技术组：拥有仓库读写权限与部署权限
- 内容组：负责内容审核，不直接维护仓库发布
- 内容创作者：本地完成内容与素材，不直接提交仓库

### 技术组内部岗位分工

- 视频服务器维护成员：负责上传视频并提供可用视频链接
- 仓库维护成员：负责将视频链接嵌入 MD、加入页面、提交并部署

## 🚀 参与流程

### 第一步：认领主题

1. 查看 [事故主题清单](docs/incidents/index.md) 中的待认领主题
2. 在组织组的「贡献记录表」中登记认领
3. 每组认领 1 个事故主题

### 第二步：完成内容创作

1. 参考 [事故页面模板](docs/incidents/_template.md) 编写故障分析文档
2. 使用 `mkdocs serve` 本地预览并截图
3. 使用 NotebookLM 生成事故分析视频

### 第三步：打包提交内容组审核

将 `项目文件 + 预览截图 + 视频文件` 打包发给内容组。

### 第四步：技术组接力上线

1. 视频服务器维护成员上传视频并给出视频链接
2. 仓库维护成员将视频链接嵌入 MD
3. 仓库维护成员完成页面加入、提交与部署

## 📝 内容规范

### 必须包含的模块

| 模块 | 说明 | 字数建议 |
|------|------|----------|
| 事故概况 | 时间、地点、规模、影响 | 信息框 + 500字 |
| 原因分析 | 技术原因、管理原因、连锁反应 | 800-1500字 |
| AI 辅助分析 | NotebookLM 分析视频 | 视频 3-10 分钟 |
| 经验与启示 | 技术、管理、对我国借鉴 | 500-800字 |
| 参考文献 | 至少 3 条，含官方报告 | - |

### 内容质量要求

✅ 必须做到：
- 事实准确，数据有来源
- 时间线清晰，因果关系明确
- 参考文献格式规范
- 语言通顺，无明显错别字

❌ 请避免：
- 直接复制粘贴大段原文（需改写）
- 使用未经核实的网络信息
- 遗漏关键技术细节
- 视频内容与文档结论不一致

## 📁 文件命名规范

### 目录命名

格式：`年份-地区关键词`（全小写，连字符分隔）

```
✅ 正确示例：
incidents/2003-us-canada/
incidents/1965-us-northeast/
incidents/2012-india/

❌ 错误示例：
incidents/2003美加大停电/     # 不要用中文
incidents/2003_US_Canada/     # 不要用下划线和大写
incidents/blackout-2003/      # 年份放前面
```

### 文件命名

- 主文件统一命名为 `index.md`
- 图片：`fig-序号-描述.png`（如 `fig-01-timeline.png`）
- 数据：`data-描述.csv`（如 `data-load-curve.csv`）

## ✍️ Markdown 写作指南

### 标题层级

```markdown
# 一级标题（页面标题，只用一次）

## 二级标题（大章节）

### 三级标题（小节）

#### 四级标题（细分内容，尽量少用）
```

### 信息框

```markdown
!!! info "提示标题"
    提示内容，用于补充说明

!!! warning "警告标题"
    警告内容，用于强调重要信息

!!! note "备注标题"
    备注内容，用于注释说明
```

### 表格

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 内容 | 内容 | 内容 |
```

### 流程图（Mermaid）

```markdown
~~~mermaid
flowchart TD
    A[开始] --> B{判断}
    B -->|是| C[结果1]
    B -->|否| D[结果2]
~~~
```

### 图片插入

```markdown
![图片描述](../assets/images/your-image.png)
```

## 🎬 AI 视频制作与嵌入

### 内容创作者阶段

1. 使用 NotebookLM 生成分析视频
2. 保存原视频文件，连同文档一起提交内容组
3. 文档中可先保留视频占位区块，不必先写最终链接

### 技术组上线阶段（仓库维护成员）

拿到视频服务器链接后，在 MD 中嵌入：

```html
<div class="video-container">
  <iframe
    src="https://你的视频服务器地址/your-video"
    title="AI 辅助分析视频"
    frameborder="0"
    allowfullscreen>
  </iframe>
</div>
```

## 📤 提交材料与交接

### 内容创作者提交给内容组（唯一通道）

必须包含：

1. 项目文件（至少包含 `docs/incidents/<case>/index.md` 与相关素材）
2. 本地预览截图（PNG/JPG）
3. NotebookLM 分析视频文件

### 内容组转交技术组

1. 内容组完成审核并给出通过结论
2. 将审核通过材料转交技术组视频服务器维护成员
3. 视频链接生成后，转交仓库维护成员完成上线

---

## ✅ 审核与上线流程

```
内容创作者本地产出（MD + 预览截图 + 视频）
   ↓
提交内容组审核
   ├─ 不通过 → 返回修改意见给内容创作者
   └─ 通过
        ↓
技术组视频服务器维护成员上传视频并生成链接
        ↓
技术组仓库维护成员嵌入视频链接、加入页面、部署上线
```

### 审核标准

| 审核项 | 通过标准 |
|--------|----------|
| 内容完整性 | 所有必需模块齐全 |
| 事实准确性 | 关键数据有文献支撑 |
| 格式规范 | 符合模板结构，Markdown 语法正确 |
| 预览可用性 | 本地预览截图清晰、页面结构完整 |
| 视频质量 | 视频可播放，内容与文档一致 |

## ❓ 常见问题

### Q: 我没有 Git 经验，怎么参与？

A: 你只需要完成本地三件套并提审：
1. `index.md` 文档
2. 预览截图
3. NotebookLM 视频

仓库提交与部署由技术组负责。

### Q: 我可以直接提交 PR 吗？

A: 当前流程下，只有技术组有仓库写权限。请走“提交内容组审核”的通道。

### Q: 找不到足够的参考资料怎么办？

A: 建议资料来源：
- 官方调查报告（IEEE、NERC、国家能源局等）
- 学术论文（知网、IEEE Xplore、Google Scholar）
- 权威新闻报道

### Q: 如何本地预览页面？

A:
```bash
pip install -r requirements.txt
mkdocs serve
# 访问 http://localhost:8000
```
