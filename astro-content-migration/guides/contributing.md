# 贡献指南

欢迎参与“电力系统大停电事故案例库”项目。本指南面向 Astro 新项目的协作流程，重点说明内容创作、审核交接与技术上线的边界。

## 角色与权限

### 当前权限边界

- 技术组：拥有仓库读写权限与部署权限
- 内容审核组：负责选题确认与内容审核
- 内容策划组：本地完成 Markdown 内容与视频素材，不直接发布站点

### 技术组内部职责

- 视频服务器维护成员：上传视频并提供可访问链接
- 仓库维护成员：接入 Markdown、嵌入视频链接、提交部署

## 参与流程

### 1. 认领主题

1. 查看内容审核组发布的主题清单
2. 在贡献记录表中登记认领，或提交自主选题等待确认
3. 每组建议认领 1 个事故主题

### 2. 完成内容创作

1. 参考 [案例模板](../content/incidents/_template.md) 创建事故 Markdown
2. 按 frontmatter 填写首页所需摘要信息
3. 完成正文撰写与参考资料整理
4. 使用 NotebookLM 生成分析视频

### 3. 提交给内容审核组

至少提交以下材料：

1. 案例 Markdown 文件
2. 相关图片或图表素材
3. NotebookLM 分析视频文件

如已进行本地预览，可附上页面截图辅助审核。

### 4. 技术组接力上线

1. 视频服务器成员上传视频并提供链接
2. 仓库维护成员补充 `video_url` 或页面嵌入配置
3. 仓库维护成员将案例接入索引、检查预览并部署

## 内容规范

### 必须包含的模块

| 模块 | 说明 |
| ---- | ---- |
| frontmatter | 首页摘要、地图点位、时间、状态等结构化字段 |
| 事故概述 | 事故背景、规模、核心结论 |
| 事故根源分析 | 直接原因与深层原因 |
| 事故时序 | 关键节点与系统演化过程 |
| 影响与后果 | 影响规模、次生影响 |
| 结构性问题 | 事故反映出的长期系统矛盾 |
| 事后改革 | 事故后的政策或技术改进 |
| 安全分析视角 | 从课程或专业知识点回看事故 |
| AI 分析视频 | 交付视频文件，页面后续嵌入 |
| 参考资料 | 至少 2 条，优先官方报告 |

### 内容质量要求

必须做到：

- 事实准确，关键数据可追溯
- 时间线清晰，因果关系明确
- 语言准确，术语统一
- 参考资料格式规范

请避免：

- 未核实信息直接入稿
- 大段复制原文而不改写
- 缺失关键数字、时间或结论
- 视频内容与文稿结论不一致

## 文件命名规范

### 案例文件

- 统一使用 `年份-地区关键词.md`
- 使用全小写英文和连字符
- 例如：`2003-us-canada.md`

### 素材文件

- 图片：`fig-01-timeline.png`
- 数据：`data-load-curve.csv`

### 建议目录

```text
src/content/incidents/
  2003-us-canada.md
public/cases/2003-us-canada/
  fig-01-timeline.png
```

## Markdown 写作建议

- frontmatter 只放结构化字段，不要塞长段正文
- 正文标题层级保持清晰，优先使用二级、三级标题
- 表格尽量简洁，复杂图表优先以图片或组件方式接入
- 如需插图，优先使用相对路径并与案例 slug 保持一致

## AI 视频接入

内容策划组阶段：

1. 生成视频并保留原文件
2. Markdown 中保留视频模块说明
3. 不必自行填写最终线上链接

技术组上线阶段：

1. 上传视频
2. 生成正式链接
3. 回填 `video_url` 或页面配置
4. 检查案例页播放效果

## 审核与上线流程

```text
内容策划组本地产出 Markdown + 视频
  ↓
提交给内容审核组
  ├─ 不通过：返回修改意见
  └─ 通过
      ↓
技术组上传视频并生成链接
      ↓
技术组接入站点并部署上线
```
