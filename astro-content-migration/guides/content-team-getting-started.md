# 内容策划组上手指南

本文档面向无仓库写权限的内容策划组同学，帮助你在 Astro 新项目路线下完成案例写作、视频制作和提审。

## 你需要完成什么

最短交付路径如下：

1. 认领一个事故主题
2. 基于模板撰写案例 Markdown
3. 整理参考资料和图表素材
4. 生成 NotebookLM 分析视频
5. 将 Markdown、素材和视频一起提交给内容审核组

## 一、先认领主题

1. 获取内容审核组发布的主题清单
2. 选择未被认领的事故主题
3. 在贡献记录表中登记认领

建议每人或每组先完成 1 个主题，优先保证质量。

## 二、获取最新模板

即使你没有仓库权限，也建议使用技术组提供的最新项目压缩包或迁移目录，确保你使用的是同一版模板和字段规范。

重点文件：

- `content/incidents/_template.md`
- `guides/contributing.md`

## 三、按模板写案例

### 文件位置建议

后续新项目中，案例文件建议放在：

```text
src/content/incidents/<slug>.md
```

示例：

```text
src/content/incidents/2012-india.md
```

### 写作顺序建议

1. 先填写 frontmatter
2. 再写事故概述和时间线
3. 补充原因、影响、改革
4. 最后统一核对数据和参考资料

### frontmatter 的作用

frontmatter 里的字段将被首页、地图、索引和案例详情页复用，所以请尽量填写完整，尤其是：

- `title`
- `slug`
- `year`
- `date`
- `region`
- `act`
- `coordinates`
- `hook`
- `affected_people`
- `duration`
- `status`

## 四、本地预览

本地预览不是硬性要求，但强烈建议在提交前看一遍页面效果。

如果技术组已经把 Astro 项目骨架搭好，可按下面方式预览：

```bash
npm install
npm run dev
```

然后访问：

```text
http://localhost:4321
```

建议重点检查：

- 标题和摘要信息是否显示完整
- 表格和图片是否排版正常
- 时间线和分节标题是否清晰
- 视频说明模块是否保留

## 五、生成 NotebookLM 分析视频

1. 准备官方报告、论文、权威新闻等资料
2. 在 NotebookLM 中生成分析视频
3. 导出并保存原视频文件
4. 确保视频内容与文稿结论一致

## 六、提交给内容审核组

请至少提交：

1. 案例 Markdown 文件
2. 图片或图表素材
3. NotebookLM 视频文件

如已做预览，可一并附上截图。

## 七、自检清单

- 文档结构完整
- 关键数据有来源
- frontmatter 字段填写齐全
- 专业术语前后一致
- 已生成视频文件
- 已提交给内容审核组

## 当前特别说明

目前项目中还没有正式收录的案例正文。你后续提交的内容将成为首批正式案例，因此更需要保证字段规范和结构统一。
